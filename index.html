VegNonVeg

Conceptual UI Design Project | Oct 2021

Description:

This is a conceptual mobile design project of vegnonveg.com.

VegNonVeg is India’s first multi-brand sneaker store, based in Delhi.

VegNonVeg carries a heavily curated selection of all kinds of sneakers ranging from classic and heritage models to their modern tech-infused counterparts, and limited edition or hard to find collaborations across brands.

They provide customers with the unique opportunity to shop multiple brands under one roof.



My Role:

Interaction Design | Visual Design



Tools & Technologies Used:

Figma | Photoshop

Problem:

Vegnonveg only has an online website and a retail store for their customers to shop multiple brands under one roof.

They don’t have a mobile app for their customers.



Solution:

This is my conceptual design of vegnonveg mobile app for android & iOS devices.

Process:

Research > Prototype > Design > Add Interactions > Test > Research > Design & Improve

October 2021 | 2 Week Project

PlaceMe

ASP.NET Web Application | May 2021 | Internship Project

Description:

Placeme is a project that I built during my internship.

The aim of this application was to provide a portal where students can get placed in their desired organisations and the organisations can hire students with the required skill set and knowledge for the job profiles.

Deliverables:

Design and build a system with basic CRUD(Create, Read, Update, Delete) functions which would consist of admin and user.

Admin would control and keep track of everything on the backend using the admin dashboard.

Users would perform a basic set of functions that a student/employee would on a job portal.



My Role:

Visual Design | UX Design | Frontend Development | Backend development



Process:

Research > Design > Code Front-end > Test > Code Back-end > Test > Get feedback > Code & Approve

How It Works:

This portal has two very important stakeholders.

First is the admin/employer and second is the user i.e student/employee.



1. Admin:

Login to the admin account using the proper credentials.

2. Admin:

View admin dashboard and manage companies, users & job applications.

3. Admin:

Add different jobs on the portal with all the mandatory details such as job profile, description, criteria, requirements, vacancies, etc.

4. Admin:

View companies and edit/delete company details.

5. User:

Login to the user account, if not an registered user then register and login.

6. User:

If not a registered user then register and login.

7. User:

Lands on the homepage.

8. User:

Edit the profile details.

9. User:

Update the qualification details.

10. User:

Browse different companies on the homepage.

11. User:

View the company info and apply for the desired position with respect to the job description, requirement and criteria.

12. Admin:

Browse different job applications.

13. Admin:

View the profile and the qualification details of the students who applied for the job.

14. Admin:

Manage the eligibility of a student with respect to the job criteria.

15. Admin:

Manage the eligible users and user schedules.

16. Admin:

Create a meet-up schedule for the students who are eligible for the job.

17. User:

View and stay up to date with the meet-up schedule for the applied job.

Timeframe:

May 2021 | 1 Week Project



Tools & Technologies Used:

Microsoft Visual Studio | Visual Studio Code | HTML/CSS | Bootstrap | JavaScript | jQuery |
Microsoft SQL Server Management Studio | C# | SQL
